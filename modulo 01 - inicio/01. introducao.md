## O que é Docker?

```
  O Docker é um software que reduz a complexidade de desde o ponto inicial do primeiro comando ou pacote da aplicação até o final do desenvolvimento das aplicações
```

```
  Onde configuramos containers, que são como servidores para aplicações
```

```
  Com facilidade, podemos criar ambientes independentes e que funcionam em diversos SO's
```

```
  E ainda deixa os projetos.
```

## Por que usar docker?

```
  O Docker proporciona mais velocidade na configuração do ambiente de um dev ou de uma AWS/Servidor conseguimos aproveitar todos os container para subir eles sem precisar configurar maquina de produção (assim ganha mais agilidade).
```

```
  Pouco tempo gasto em manutenção, containers são executados como configurados
```

```
  Performance para executar aplicação (podendo escolher as versões necessária), mais performático que uma VM (Virtual Machine - Máquina virtual -> é um ambiente virtualizado que funciona como um sistema de computação com sua própria CPU, memória, interface de rede e armazenamento.)
```

```
  Nos livra da "Matrix from Hell" 
```

## O que é Matrix from Hell?

```
Quando temos um problema em simular ou rodar o projeto em diversas maquinas, pois, cada projeto pode ter configurações e/ou linguagens diferentes.

Mas com o docker, apenas subindo os containers já agiliza tudo isso.
```

## Qual versão utilizar?

```
O Docker é dividido em duas versões: CE x EE
```

```
CE é a Community Edition (edição da comunidade), edição gratuita, que nos possibilita utilizar o Docker normalmente.
```

```
EE é a Enterprise Edition, edição paga, há uma garantia maior das versões que são disponibilizadas e você tem suporte do time do Docker.
```
